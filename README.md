# factum.nps

NPS-приложение для сбора обратной связи с интеграцией Битрикс24.

## Быстрый старт

```bash
npm install
npm run dev
```

Откройте http://localhost:5173

## Структура проекта

```
src/
├── data/index.js          # Мок-данные + Bitrix24 API сервис
├── hooks/store.jsx        # Глобальный стейт (useContext + useReducer)
├── components/
│   ├── ui.jsx             # UI-компоненты (ScoreCircle, Badge, Button...)
│   ├── Topbar.jsx         # Навигация
│   └── AddModal.jsx       # Модалка добавления записи
├── pages/
│   ├── Dashboard.jsx      # Дашборд с аналитикой и таблицей
│   ├── Projects.jsx       # Все проекты с фильтрами по отделу
│   └── Calls.jsx          # Экран обзвона для менеджера
└── App.jsx                # Роутинг
```

## Интеграция с Битрикс24

### 1. Создайте входящий вебхук

В Битрикс24 → Приложения → Вебхуки → Входящий вебхук.

Нужные права:
- `crm` — чтение и запись CRM
- `user` — список пользователей

### 2. Создайте пользовательские поля в CRM

В CRM → Настройки → Пользовательские поля сделки:
- `UF_CRM_NPS_SCORE` — Число (оценка NPS 1-10)
- `UF_CRM_NPS_COMMENT` — Строка (комментарий)

### 3. Добавьте .env файл

```
VITE_BITRIX_WEBHOOK=https://factum.bitrix24.kz/rest/638/0v78pcs5dhrlnkne
```

Без этой переменной приложение работает с мок-данными.

### 4. Настройте маппинг полей

Откройте `src/data/index.js` → функция `fetchDeals()`.
Настройте маппинг полей Битрикс под ваши поля:
```js
service: deal.TYPE_ID ?? '',           // тип сделки = услуга
specialist: deal.ASSIGNED_BY_ID ?? '', // ответственный
company: deal.COMPANY_TITLE ?? '',     // компания
```

### Как работает синхронизация

1. **Загрузка** — кнопка "Битрикс24" в шапке подтягивает сделки через `crm.deal.list`
2. **Сохранение оценки** — после сохранения NPS вызывается `crm.deal.update` с полем `UF_CRM_NPS_SCORE` + добавляется комментарий в ленту активности
